# 需求文档索引

本项目将需求拆分为两份文档，便于产品与技术分别维护、对齐验收。

| 文档 | 说明 |
|------|------|
| [**need-product.md**](./need-product.md) | **产品需求（PRD）**：定位、用户、场景、痛点、能力描述、体验与异常验收、MVP 与成功指标 |
| [**need-tech.md**](./need-tech.md) | **技术需求（TRD）**：架构、数据与 chunk 元数据、API 契约、检索与 Prompt 约束、非功能与安全、测试要点 |

**阅读顺序建议**：先 PRD 锁定「做什么、验什么」，再 TRD 锁定「怎么做、测什么」。
